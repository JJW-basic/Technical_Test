[ Background ]
1. 사용자가 URL 주소와 키워드를 입력하면 키워드에 포함된 정보를 수집하고 정리해 주는 기능을 구현하고자 함.
2. window OS에서 Antigravity IDE의 AI agent를 이용한 바이브 코딩으로 작업을 진행.
3. 01-system-rules.md 파일에 프로젝트 진행 방향성을 명시했음.
4. docker compose up, http://localhost/, http://localhost/api/health 정상 동작 확인
5. 웹 스크랩핑 시 전달받을 URL, 키워드 정보를 받기 위한 UI는 구현됨.
6. GPT-OSS 120B (Medium) 모델로 아래의 내용(User Command)을 구현 요청했으나, 'scraper.py' 생성 외에 동작하지 않음.
	- 해당 모델의 한계로 보이며 .agents\rules\01-system-rules.md에 명시한 내용도 수행하지 않음.


[ User Command ]
웹 스크랩핑 기능을 구현하고자 한다.
웹 스크랩핑 시 이미지는 제외하도록 한다.
사용자가 URL, 키워드 정보를 입력해야 데이터 분석, 정리를 진행하는 현재의 구조를 유지한다.
데이터 분석, 정리가 끝난 이후 아래에 '분석 결과 리포트'를 보여주는 현재의 구조를 유지한다.

사용자가 '분석 요청하기' 버튼을 클릭하면 아래의 동작이 있어야 한다.
1. URL, 키워드 정보가 제대로 전달되었는지 확인한다.
	- 둘 중 하나라도 누락이 되었다면, 사용자에게 '정상적인 접근이 아닙니다.'라고 명시한 이후 처음 페이지로 보낸다.
		- 당연히 아래의 동작들은 수행하지 않는다.
2. URL, 키워드 정보가 제대로 전달되었다면 URL 주소 내의 도메인 주소를 기준으로 'robots.txt', 'humans.txt', 'security.txt', 'sitemap.xml' 파일의 존재 여부를 확인한다.
	- 별도의 제한 사항이 없다면 스크랩핑을 진행한다.
		- 내부에서 코드가 동작하지만 UI/UX에 변함이 없으면, 사용자는 서비스가 멈춘 걸로 판단할 여지가 있다.
		- 스크랩핑이 끝날 때까지 현재 작업이 진행 되고 있음을 UI/UX로 표현해야 한다.
		- 내부 경로에 해당 URL의 호스트명을 이름으로 하는 폴더를 생성한다.('www.naver.com'의 경우 폴더 이름은 'naver'가된다.)
	- 해당 도메인에서 데이터 수집 자동화를 허용하지 않는다면, 사용자가 웹페이지에 입력한 URL 주소를 지운 후 '해당 요청을 (도메인주소)에서 거부합니다.'라고 명시한다.
		- (도메인 주소)에는 사용자가 입력한 URL 주소 내의 도메인 주소를 입력해 주어야 한다.
		- 당연히 아래의 동작들은 수행하지 않는다.
	- 해당 도메인에서 데이터 수집 자동화는 허용하지만 접근을 거부를 명시한 내부 경로였다면, 사용자가 웹페이지에 입력한 URL 주소를 지운 후 '(도메인주소)의 보안 정책상 허용되지 않는 URL 경로입니다.'라고 명시한다.
		- (도메인 주소)에는 사용자가 입력한 URL 주소 내의 도메인 주소를 입력해 주어야 한다.
		- 당연히 아래의 동작들은 수행하지 않는다.
3. 웹 스크래핑이 끝나면 스크래핑 대상 URL의 호스트명으로 된 폴더 내에 마크다운 파일을 생성한다.
	- 파일 이름은 'scrap_(호스트명)_(넘버).md' 형식으로 만든다.('www.naver.com'의 경우 'scrap_naver_01.md'이 파일명이 된다.)
		- 파일 이름의 (넘버)는 기존 파일이 아무것도 없는 경우 '01'을 부여하고 기존 파일이 존재한다면 그 다음 숫자를 부여한다.('scrap_naver_02.md' 파일이 존재한다면 현재 생성된 파일 이름은 'scrap_naver_03.md'이 된다.)
	- 생성된 마크다운 파일엔 스크래핑한 정보를 입력한다.
4. 사용자가 입력한 키워드가 새로 생성된 마크다운 파일에 존재하는지 확인한다.
	- 영문의 경우 대소 구분 없이 탐색한다.
	- 키워드가 확인된 경우 동일한 키워드가 존재하는 만큼 숫자를 카운팅 한다.
5. '분석 결과 리포트'를 사용자에게 보여준다.
	- 사용자가 입력한 키워드가 몇 번 등장했는지 명시한다.
	- 현재 스크래핑 작업으로 생성된 마크다운 파일의 내용을 보여준다.
		- 사용자가 입력했던 키워드가 있었다면 내용에서 키워드를 볼 수 있도록 글자 색, 크기를 다른 내용들과 다르게 하는 방식으로 강조해줘야 한다.
